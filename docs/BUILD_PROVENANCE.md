# Build Provenance

This document explains how AegisOps Autopilot satisfies the Devpost New & Existing project rule with a reviewable implementation trail.

## What Judges Should Inspect

Run:

```bash
pnpm run provenance:audit
```

The command writes `reports/build_provenance.md` from the local Git history and checks that major contest-delivery milestones have matching source, documentation, or report evidence.

## Covered Milestones

- Track 4 AegisOps submission workspace and Devpost copy.
- Public runnable demo path through StackBlitz and GitHub Pages.
- Judge quickstart, rubric scorecard, and evidence packet.
- Alibaba Cloud deployment proof path and verifier.
- Qwen integration audit and model-ops evidence.
- Qwen Function Calling tool schemas, live tool-call loop, and active-incident tool scoping.
- Visible rubric evidence UI for the 30/30/25/15 review path.
- Upload-ready demo video, captions, chapters, and video asset audit.

## Claim Boundary

This provenance record proves substantial repository work for the submitted Qwen Cloud Hackathon project. It does not claim live Alibaba deployment or optional blog publication are complete; those remain account-gated until performed by the account owner.
